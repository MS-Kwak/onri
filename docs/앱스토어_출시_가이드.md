# 온리 앱 스토어 출시 전체 가이드

코딩은 이미 끝난 상태입니다. 아래는 **앱을 실제 스마트폰에서 다운로드할 수 있게 만드는** 전체 과정입니다.

---

## 1단계: 개발자 계정 만들기

### Apple (iOS 앱스토어)

1. https://developer.apple.com 접속
2. Apple ID로 로그인 (없으면 새로 만들기)
3. "Apple Developer Program" 가입 클릭
4. 연간 **$99 (약 13만원)** 결제
5. 승인까지 **24~48시간** 소요

### Google (Play 스토어) ✅ 완료

1. https://play.google.com/console 접속
2. Google 계정으로 로그인
3. "개발자 계정 만들기" 클릭
4. 일회성 **$25 (약 3만3천원)** 결제
5. 본인인증 (신분증 제출) — 승인까지 **2~7일** 소요
6. ✅ Android 휴대기기 액세스 확인
7. ✅ 연락처 전화번호 인증

---

## 2단계: Firebase에 앱 등록하기

Firebase Console (https://console.firebase.google.com) → onri 프로젝트(`onri-b8b36`)에서:

### Android 앱 추가

1. 프로젝트 설정 → "앱 추가" → Android 선택
2. 패키지 이름: `com.onri.app` 입력
3. "앱 등록" 클릭
4. `google-services.json` 파일 다운로드
5. 다운로드한 파일을 `onri-app/` 폴더에 넣기

### iOS 앱 추가

1. 같은 화면에서 "앱 추가" → iOS 선택
2. 번들 ID: `com.onri.app` 입력
3. "앱 등록" 클릭
4. `GoogleService-Info.plist` 파일 다운로드
5. 다운로드한 파일을 `onri-app/` 폴더에 넣기

---

## 3단계: 앱 아이콘 / 스플래시 이미지 준비

아래 파일들을 디자인해서 `onri-app/assets/` 폴더에 넣어야 합니다:

| 파일                          | 크기        | 설명                                      |
| ----------------------------- | ----------- | ----------------------------------------- |
| `icon.png`                    | 1024x1024px | 앱 아이콘 (둥근 모서리 없이, 자동 적용됨) |
| `splash-image.png`            | 1284x2778px | 앱 열 때 보이는 로딩 화면                 |
| `android-icon-foreground.png` | 1024x1024px | 안드로이드 적응형 아이콘 전경             |
| `notification-icon.png`       | 96x96px     | 푸시 알림 아이콘 (흰색+투명 배경)         |

## 4단계: 앱 빌드하기

터미널에서 실행:

```bash
cd onri-app
npx expo prebuild
```

### Android APK/AAB 빌드

```bash
npx eas build --platform android
```

### iOS 빌드 (Mac 필요)

```bash
npx eas build --platform ios
```

> 처음 실행하면 EAS 계정 로그인과 빌드 설정을 안내해줍니다. 무료 플랜으로 가능합니다.

---

## 5단계: 스토어에 앱 정보 등록

### Google Play Console

1. https://play.google.com/console 로그인
2. "앱 만들기" 클릭
3. 아래 정보 입력:
   - **앱 이름**: 온리 - 결이 맞는 사람과의 연결
   - **기본 언어**: 한국어
   - **앱 유형**: 앱
   - **무료/유료**: 무료 (인앱 결제 있음)
4. "스토어 등록정보" 작성:
   - 간단한 설명 (80자)
   - 자세한 설명 (4000자)
   - 스크린샷 (핸드폰 캡처 최소 2장)
   - 그래픽 이미지 (1024x500px 배너 1장)
5. "콘텐츠 등급" 설문 작성 (앱 내용 관련 질문에 답하면 자동 등급 부여)
6. "개인정보처리방침 URL" 입력: `https://onri.vercel.app/privacy`

### App Store Connect (Apple)

1. https://appstoreconnect.apple.com 로그인
2. "나의 앱" → "+" → "신규 앱"
3. 아래 정보 입력:
   - **이름**: 온리
   - **기본 언어**: 한국어
   - **번들 ID**: com.onri.app
   - **SKU**: onri-app
4. 앱 정보 작성 (Google과 비슷):
   - 설명, 스크린샷 (6.7인치, 6.5인치 각각 필요), 앱 아이콘
   - 개인정보처리방침 URL
   - 연령 등급 설문
   - **앱 내 구매 설정** (하트 충전 상품 등록)

## 6단계: 빌드 파일 업로드 + 심사 제출

### Google Play

1. Play Console → 프로덕션 → "새 버전 만들기"
2. 4단계에서 만든 `.aab` 파일 업로드
3. 출시 노트 작성 (예: "온리 첫 출시")
4. "심사를 위해 제출" 클릭
5. 심사 기간: **보통 1~3일** (첫 앱은 최대 7일)

### App Store

1. 4단계 빌드가 자동으로 App Store Connect에 올라감
2. App Store Connect → 빌드 선택
3. "심사를 위해 제출" 클릭
4. 심사 기간: **보통 1~2일** (첫 앱은 최대 7일, 거절될 수 있음)

---

## 7단계: 심사 통과 후

- **승인되면** → "출시" 버튼 클릭 → 스토어에 공개
- **거절되면** → 거절 사유 확인 → 수정 후 재제출

---

## 전체 타임라인 요약

```
1일차: 개발자 계정 가입 + Firebase 앱 등록
       ↓ (계정 승인 대기 1~7일)
승인 후: 앱 아이콘/스크린샷 준비
         앱 빌드 (1~2시간)
         스토어 정보 등록 (2~3시간)
         심사 제출
         ↓ (심사 대기 1~7일)
승인 후: 출시 버튼 → 스토어 공개
```

**최단 시나리오: 약 1주일** / **여유 있게: 약 2주일**

## 프로젝트 구조 참고

```
onri-app/                       ← React Native (Expo) 프로젝트
├── App.tsx                     ← WebView + FCM + 딥링크 + 뒤로가기
├── app.json                    ← 앱 설정 (bundle ID, 권한, Firebase 플러그인)
├── google-services.json        ← (Firebase에서 다운로드 후 배치)
├── GoogleService-Info.plist    ← (Firebase에서 다운로드 후 배치)
└── assets/                     ← 아이콘/스플래시 (교체 필요)

onri/src/                       ← 기존 Next.js 프로젝트
├── app/api/fcm-token/route.ts  ← FCM 토큰 저장/삭제 API
└── lib/native-bridge.ts        ← 네이티브 앱 감지 + 토큰 등록 헬퍼
```

## App.tsx에 포함된 기능

- **WebView** — `onri.vercel.app` 로드
- **Android 뒤로가기** — WebView 히스토리 뒤로 or 앱 종료 확인
- **FCM 푸시** — 권한 요청 → 토큰 발급 → 포그라운드 알림 표시
- **딥링크** — 푸시 클릭 시 `data.path`로 WebView 이동
- **외부 링크** — 카카오/구글/Apple 로그인, PortOne은 WebView 내, 나머지는 시스템 브라우저
- **파일 업로드** — `allowFileAccess` 등 활성화 (셀카/갤러리)
